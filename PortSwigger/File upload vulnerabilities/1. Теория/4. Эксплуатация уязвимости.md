С точки зрения безопасности самый худший сценарий возникает тогда, когда сайт позволяет загружать серверные сценарии, такие как файлы PHP, Java или Python, и одновременно настроен на исполнение этих файлов в качестве кода. Это дает злоумышленнику легко создать свою собственную оболочку на сервере.

**Веб-оболочка**
Веб-оболочка представляет собой вредоносный скрипт, позволяющий атакующему исполнять произвольные команды на удаленном веб-сервере путем отправки соответствующих HTTP-запросов нужному адресу.
При успешной загрузке веб-оболочки вы получаете полный контроль над сервером. Это означает, что вы сможете читать и записывать любые файлы, извлекать чувствительные данные, а также использовать сервер для организации дальнейших атак против внутренней инфраструктуры или других серверов вне сети. 
(Пример PHP-кода, который используется для чтения любых файлов из файловой системы)
*<?php echo file_get_contents('/путь/до/целевого/файла'); ?>*
(Более гибкая версия веб-оболочки)
*<php echo system($_GET['command']); ?>*
Данный скрипт позволит передать произвольную системную команду, например:
GET /example/exploit.php?command=id HTTP/1.1
